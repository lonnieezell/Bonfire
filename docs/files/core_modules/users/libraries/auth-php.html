Auth

Provides authentication functions for logging users in/out, restricting access to controllers, and managing login attempts.

Security and ease-of-use are the two primary goals of the Auth system in Bonfire.  This lib will be constantly updated to reflect the latest security practices that we learn about, while maintaining the simple API.

Author

Lonnie Ezell

restrict()

public function restrict($permission = null,
$uri = null)

Checks that a user is logged in (and, optionally of the correct role) and, if not, send them to the login screen.

If no permission is checked, will simply verify that the user is logged in.  If a permission is passed in to the first parameter, will check the user’s role and verify that role has the appropriate permission.

Parameters

$permission(Optional) A string representing the permission to check for.
$uri(Optional) A string representing an URI to redirect, if false

Return

trueif the user has the appropriate access permissions.
redirectto the previous page if the user doesn’t have permissions.
redirect’/login’ page if the user is not logged in.
